=== DynamicObstacleManager === 
Он будет добавлять движущиеся препятствия:
	люди / животные / другой робот
Логика:
	1. препятствие имеет позицию
	2. каждый tick может сдвинуться
	3. GlobalPlanner учитывает dynamic obstacles
	4. Renderer показывает их другим цветом
	5. если путь заблокирован → replanning
То есть сначала не усложняем нейросеть, а добавляем динамические объекты в среду и заставляем A* перепланировать путь.
=========
Теперь уже есть:
	dynamic partially observable world
И это очень серьёзный уровень сложности.
Сейчас система уже умеет:
	исследовать
	запоминать
	планировать
	оптимизировать энергию
	избегать динамических препятствий
	перепланировать маршрут
	анализировать replay
	сравнивать алгоритмы
то уже не “бот для капусты”, а настоящий автономный navigation stack.

=== Replanning / Path Invalidation ===
Сейчас A* строит путь, но если:
	dynamic obstacle
		↓
	перекрыл путь
агент может:
	упереться
	стоять
	делать лишние шаги
Нужно научить его:
	видеть что путь стал невалидным
		↓
	мгновенно перестраивать маршрут
